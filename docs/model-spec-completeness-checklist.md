# Model Specification Completeness Checklist

This checklist is the publication-readiness control surface for the Amor Fati
model specification. It links each major model family to the current notation,
rules, implementation anchors, output surfaces, SFC/ledger mapping,
calibration evidence, and validation coverage.

It does not define new model behavior. The canonical overview remains
[model-specification.md](model-specification.md), while detailed equations,
ODD/ODD+D notes, SFC artifacts, calibration evidence, and validation ownership
remain in their source documents.

## Status Legend

| Status | Meaning |
| --- | --- |
| `Complete` | A reviewer can follow the current implemented model from linked docs and anchors without needing new source material. |
| `Partial` | A reviewable surface exists, but publication-grade consolidation, empirical grounding, or coverage is still incomplete. |
| `NA` | The column is not meaningful for that family. |
| `Missing` | No reviewable source exists. This should be treated as a paper-readiness gap. |

No row below should hide uncertainty. `Partial` is acceptable when it points to
a concrete source, limitation, or follow-up family.

## Open Publication-Readiness Work

The current top-level documentation has no `Missing` model-family row. The main
remaining issue is depth: several implemented families are documented enough to
be reviewable, but still need publication-grade consolidation.

Priority order for the remaining milestone work:

1. #709: banking and financial-sector equation consolidation.
2. #708: stochastic processes and randomness contract.
3. #710: SFC matrices to model equations.

## Review Rule

When adding a new model mechanism or publication-facing equation, update this
checklist if it changes any of the following:

- notation coverage;
- rule-family coverage;
- implementation anchor ownership;
- output-column visibility;
- SFC/ledger mapping;
- calibration or empirical evidence;
- validation or diagnostic ownership;
- known limitations or publication-readiness gaps.
