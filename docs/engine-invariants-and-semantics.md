# Engine Invariants And Economic Semantics

This index is the reviewer-facing entry point for Amor Fati's engine law. It
does not replace the detailed model, calibration, matrix, or diagnostics
documents. It routes each major invariant or economic semantic to its source of
truth, enforcement point, test or diagnostic coverage, and failure severity.

Use this file when deciding whether a regression belongs in PR CI, integration
tests, nightly health summaries, stress diagnostics, calibration review, or
known-limitation tracking.

## Classification Vocabulary

| Class | Meaning | Default failure effect |
| --- | --- | --- |
| `hard_invariant` | Accounting, runtime, schema, or impossible-state contract that must always hold. | Fail PR CI, integration tests, nightly diagnostics, or the relevant export. |
| `normal_path_expectation` | Expected behavior for baseline or normal-validation profiles at the current 60-month operational horizon. | Fail only when encoded as a normal-path health summary or integration gate; otherwise review as model-health evidence. |
| `stress_or_exploratory_diagnostic` | Deliberate stress channel or research diagnostic without stable acceptance thresholds. | Report or warn; do not reinterpret expected stress outcomes as baseline failures. |
| `calibration_warning` | Empirical or stylized target band that should guide calibration but is not an accounting error. | Warn and document; promote to hard fail only with a written threshold rationale. |
| `known_limitation` | Intentional gap, unsupported stock family, bridge assumption, or currently unowned flow. | Keep visible and auditable; do not silently balance or hide it. |

## Severity Routing

| Severity | Typical owner | Notes |
| --- | --- | --- |
| Hard fail | Unit/property tests, integration tests, generated-output checks, nightly runner, SFC validation, health summary hard metrics. | Used for ledger conservation, SFC exactness, missing required artifacts, malformed TSVs, impossible stock states, and invalid active-bank invariants. |
| Warning | Nightly health summary, diagnostics reports, empirical validation, calibration docs. | Used for soft calibration bands, material drift, or incomplete empirical bridges. |
| Report only | Scenario, stress, robustness, lead-lag, loan-quality, profiling, and exploratory diagnostics. | Evidence for investigation, not a claim that baseline calibration failed. |
| Known limitation | ADRs, data bridge, matrix mapping, calibration register, this index. | Must remain explicit until implemented, rejected, or converted into a supported contract. |

## Minimal Hard-Fail Set

The following should remain hard failures in every profile or test layer where
the required artifacts exist:

- ledger conservation or runtime ledger execution failure;
- SFC exactness failure;
- missing required seed/month rows for a completed profile;
- malformed required TSV or manifest artifact;
- stale committed generated outputs;
- unsupported flow direction in a semantic projection;
- impossible negative stock where the domain contract says non-negative;
- invalid active-bank invariant;
- material bank-capital waterfall or reconciliation residual;
- current-account closure residual.

## Normal-Path Signals

The following are not universal accounting laws, but they are important
normal-path health signals for the current five-year operational horizon:

- ordinary baseline bank failures and all-failed fallback evidence;
- employment, unemployment, wage, demographic, inflation, and price-pressure
  paths;
- total credit to GDP, default/NPL ratios, and household liquidity stress;
- GDP proxy direction and terminal decline;
- debt/GDP, deficit/GDP, current-account/GDP, and private-credit renewal;
- bank opening balance-sheet guardrails;
- household and firm loan-origination quality cohorts.

Promotion from warning/report to hard failure requires a documented threshold
rationale in the comparison or diagnostics contract.

## Known Limitations That Must Stay Visible

- Bank capital is not holder-resolved equity. It is persisted bank regulatory
  state and SFC-validated diagnostic state.
- The unretained share of bank gross income currently has no modeled
  owner-side receiver.
- Some symbolic matrix rows intentionally expose unsupported or incomplete
  holder coverage rather than balancing by residual.
- Empirical source extraction is not yet fully manifest-backed for several
  domains, especially household microdata, firm demography, holder-resolved
  financial accounts, and scenario-specific shocks.
- Performance budgets are soft warning reports; hard gates require documented
  promotion after comparable `main` runs.

## Maintenance Rule

When adding or changing an invariant, diagnostic target, scenario class,
supported asset family, or semantics-bearing output column:

1. Update the detailed source document or code-level contract first.
2. Add or update the enforcing test, diagnostic, generated artifact, or health
   summary rule.
3. Update this index with the class, source of truth, enforcement point,
   coverage, and severity.
4. If the entry can fail CI or nightly diagnostics, include the threshold or
   invariant rationale in the relevant detailed document.
